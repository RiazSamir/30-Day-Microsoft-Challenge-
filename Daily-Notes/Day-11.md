# Day 11 - Email Security Policies with Microsoft Defender for Office 365

## Objective:
- Understand the email security defensive layer within Microsoft Defender for Office 365
- Review and analyse default and preset email security  policies
- Learn how Microsoft protects against Phishing, Email, and Spoofing attacks

## Platform / Tools
- Microsoft Defender XDR portal
- Email & Collaboration

## Actions Performed:
- Reviewed the Threat found in Email & Collaboration > Policies  & Rules  > Threat Policies (*Figure 1*)
- Reviewed the Default Policies found in Threat Policies (i.e., Anti-Phishing, Anti-Spam, Anti-Malware, Safe-Attachments, Safe-Links) (*Figure 2*)
- Reviewed the Templated policies, more importantly, the Preset Security Policies (*Figure 3*)
  - Preset Security Policies are essential to help your organisation have a good security posture with Microsoft's recommended Security Controls and Settings
- Reviewed the Configuration Analyser, which is used to identify our current policy configuration to improve our Security Posture. 

## SOC Relevance
- Email remains the primary initial Access for real-world Attacks
- Defender for Office 365 provides a Layered protection against phishing, malware, and impersonation
- Preset and Custom policies help enforce consistent security controls across the tenant
- Configuration Analyser highlights misconfigurations that increase risk
- These controls directly support phishing detection, Threat Hunting, and Incident response workflows.
